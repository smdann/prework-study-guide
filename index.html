HTML

HTML stands for Hypertext Markup Language.

  • HTML is used to create the structure of a webpage.
  • It's made up of elements which work together to provide content and the basic framework for all pages.

HTML Elements

  • An element is a type of content on a webpage.
  • HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets (<>).
  • Commonly used elements include: text elements, image elements, header elements, and list elements.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

CSS stands for Cascading Style Sheets.

  • CSS is a programming language used by developers to define how webpage content is presented to users.
  • CSS defines the colors, aesthetics, visual layout, and organization.

CSS Box Model

  • CSS box model: each box consists of margins outside the element, the elements borders, borders, padding, and content.
  • Every HTML element is essentially a box.
  • A margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.

CSS Rules

  • CSS rule format: selector{property: value; property: value}
  • {property: value} = declaration
  • {property: value; property: value} = declaration block
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property

Additional Info

  • The "*" selector applies a rule to all elements visible on the page.
  • Two selectors can be combined into a single CSS rule by separating them with a comma, which allows the declarations to affect both selectors.
  • Grouping elements that share the same style together helps us write Do Not Repeat Yourself (DRY) code.
  • When we create an external CSS file, we must link it to the HTML file.

Git

What is Git?

  • Git is a version control system that lets you manage and keep track of each version of your project. We access Git via the command line.
  • Git is great for tracking changes locally on your own computer, but to collaborate with others, you need to use another tool called GitHub.

Git Commands

  • Git commands always start with git
  • git status: checks what branch we are currently on, shows modified files in working directory (staged for your next commit)
  • git checkout -b feature/branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working brach to the staging area (-A flag indicates that we want to add all changes)
  • git commmit -m "descriptive message": commits your staged content as a new commit snapshot
  • git pull origin main: checks if our local branch is in sync with the base branch in GitHub and pulls down any changes
  • git push origin feature/branch-name: pushes the changes we have locally to our remote GitHub branch

Git Flow

  1. Make sure we are on the correct branch (feature/branch-name) from the command line.
  2. Add the changed files in our working directory to the staging area of Git (git add -A).
  3. Commit to our repository (git commit -m "descriptive message").
  4. Make sure current branch is up to date before opening a pull request (git pull origin main).
  5. Push up all of our changes (git push origin feature/branch-name).
  6. Naviage to appropriate repository on GitHub and open a new pull request.
  7. Set the base branch and comparison branch. Add a title and short description of changes made.
  8. Merge pull request or request review from team.
  9. Close issue if applicable.
  10. Update local branch by by entering (git checkout main) into the command line.
  11. Pull the remote main branch into the local main branch by entering (git pull origin main).

JavaScript

What is JavaScript?

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites.
  • It allows you to power your websites with effects, interactivity, data communication, and more.

Variable

  • A variable is a named container that allows us to store data in our code.
  • To declare a variable, we use the var keyword.
  • After the var keyword, we give the variable a name. This must be a unique name because it's how we'll reference the information stored in the variable in our code.
  • Finally, we use an assignment operator = to assign the specific data that we want stored in our variable.
  • Example: var topics = "HTML, CSS, Git, JavaScript";
  • The topics variable contains a string that includes all of the topics that we're studying (HTML, CSS, Git, JavaScript).
  • All strings are surrounded by quotation marks.
  • Each line of JavaScript code ends with a semicolon.

console.log

  • The word console refers to a test environment that developers use to check out their code.
  • .log() is a method which means a set of instructions that can be executed by a computer.
  • The .log() method will output whatever we add inside the parentheses to the console.
  • For the variable above, entering console.log(topics); will output the value of the variable topics to the web console because we've added it as an argument inside the parentheses.
  • To open Chrome DevTools to view console.log output: Command+Option+I

Control Flow

  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • Control flow can be interrupted by conditional statements and loops.

Conditional Statements

  • An if statement is a way of introducing decision-making into our code.
  • When using the if statement, code will only execute if the statement meets a condition, or is true.
  • if (condition) { //block of code to be executed if condition is truthy }
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.

Arrays

  • An array is a single variable that is used to hold a group of data; typically data that is related in some way.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types.
  • var studentInfo = ["Lu", 54, true]
  • Each individual data item inside an array is is assigned a unique number, called an index (which starts at 0), that's used to identify the location of the data item inside the array.
  • To access only one item within an array: console.log(variable[index];

for Loops

  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A for loop contains three important statements.
    1. The first statement determines the starting point for our loop. To start at the beginning of the array, we declare a variable x and give it a value of 0.
      • var x = 0
    2. The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. We use the length built-in array property to express that condition.
      • x < variable name.length
    3. The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increased by 1 each time. The following code is a shortcut way of writing x+1:
      • x++

Functions

  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • Functions do not automatically execute when the JavaScript file is run. You must call a function by name in order to execute it.
  • To create a function named helloWorld you would use: function helloWorld() { //code block }
  • Math function
    • To randomly pick an item from an array, we can use math functions to select a random number.
    • Math.random()
